# Centauri OS: External Platform Integrations

Integrating external platforms like OpenHands and Aider requires a modular, state-driven approach. Each integration must be encapsulated as a **Node** and adhere to the Centauri Interlock Standard.

## 1. OpenHands Integration (Modular Node)

OpenHands (formerly OpenDevin) acts as an autonomous software engineer. To integrate it into the Centauri ecosystem:

- **State Verification**: Before triggering OpenHands, the Node must verify the project path and current system state in `caroline_neuro_memory.json`.
- **NeuroRank Check**: High-priority engineering tasks (NeuroRank > 7) should be prioritized for autonomous execution.
- **Closed-Loop Feedback**: OpenHands results (PRs, code changes, or logs) must be formatted as a JSON payload and broadcasted to the `Command_Router`.

## 2. Aider Integration (Modular Node)

Aider is a CLI tool for AI pair programming. To integrate it:

- **Node Pattern**: Create a Python wrapper that executes Aider commands via `subprocess`.
- **Context Injection**: Use the `caroline_neuro_memory.json` state to determine which files Aider should focus on.
- **Broadcast**: After Aider completes a git commit or code modification, the Node must report the changes back to the system bus.

## 3. Connector Schema Standard

All connectors should use a consistent payload format for broadcasting to the `Command_Router`.

| Field | Description | Format |
|-------|-------------|--------|
| `node_id` | Unique ID for the connector (e.g., `aider_connector`) | String |
| `timestamp` | ISO 8601 timestamp of completion | ISO String |
| `status` | `SUCCESS`, `ERROR`, or `PARTIAL_SUCCESS` | Enum |
| `payload` | Data generated by the external tool (e.g., logs, file paths) | JSON Object |
| `error` | Detailed error message if status is `ERROR` | String |

## 4. Environment Management

- **API Keys**: Store sensitive keys in environment variables (e.g., `RUNPOD_API_KEY`, `OPENHANDS_TOKEN`).
- **State File**: All connectors **must** have read access to `caroline_neuro_memory.json` to maintain synchronization with the Centauri OS core.
